# 小说健康度检查清单

> 人工诊断工具。翻看最近5-10章，用直觉回答以下6个问题。
> 不是脚本——没有 PASS/FAIL。只有"作者的感觉"。
> 触发：用户说"诊断小说"/"检查健康度"/"novel doctor"。

---

## 1. 段落呼吸

**问题**：翻一遍章节，段落长度看起来自然吗？

**检查方法**：
- 快速滚动翻看最近5章
- 有没有连续5段以上的段落都在200-250字区间？（"平头"现象）
- 有没有连续多段都是500字以上的超长段？（读者跳读高发区）
- 有没有连续10段以上都是1-2行的超短段？（碎片化）

**直觉验证**：找一个你没读过这章的空白时间，打开最近一章，只看段落长度的"形状"，不看内容。形状看起来有变化吗？

**行动建议**：
- 平头 → 合并2-3段或拆散1-2段，打破均匀分布
- 超长段密集 → 找自然断点拆分
- 碎片化 → 合并相关短段

---

## 2. 对话/叙述平衡

**问题**：对话和叙述的比例读起来舒服吗？

**检查方法**：
- 翻看最近3章
- 有没有某一章对话占了80%以上——读完感觉"全是人在说话，不知道他们在哪"？
- 有没有某一章叙述占了90%以上——读完感觉"全是描述，没人说话"？
- 不需要精确数字。翻一遍，感觉如何？

**直觉验证**：对话密集章 → 闭上眼睛。你能看到角色们在什么地方说话吗？不能 → 对话缺少锚定。叙述密集章 → 读完你还能记住角色在做什么吗？不能 → 缺少行动推进。

**行动建议**：
- 对话过多 → 插入环境锚定和角色动作。见 `writing-techniques.md` §2。
- 叙述过多 → 插入对话或内心独白。或者确认：这些叙述真的需要吗？

---

## 3. 感官切换

**问题**：有没有连续好几页都只用了一种感官？

**检查方法**：
- 快速翻看最近3章，注意每页的感官类型
- 有没有连续2000字以上只用了视觉？
- 有没有某种感官（特别是触觉和嗅觉）几乎完全缺失？

**直觉验证**：读完一章后，你"记得"什么感官？只有画面 → 缺少触觉和声音。只有对话 → 缺少触觉和环境。

**行动建议**：
- 视觉主导超过2000字 → 找一个位置插入触感/声音/气味
- 特定感官缺失 → 确认是否是有意的（有些章节确实不需要某种感官），如果无意，找位置插入
- 参考 `writing-techniques.md` §1 感官与情绪的对应

---

## 4. 情绪曲线

**问题**：本章的情绪有没有起伏？

**检查方法**：
- 翻看最近3章，大致画出每章的情绪走向（高/中/低）
- 有没有某一章从头到尾维持在同一情绪调上？（全章高压/全章低压/全章中性）
- 最近几章的情绪曲线是否在重复同一种形状？

**直觉验证**：读完一章后，你有"情绪变化"的感觉吗？还是读完了觉得"很平"？

**行动建议**：
- 全章高压 → 需要插入至少一个喘息点（即使只有两行环境描写）
- 全章低压 → 需要插入至少一个冲突或担忧（即使还没有实际行动）
- 曲线重复 → 下一章有意选择不同的节奏模型（上升型 vs 下降型），见 `rhythm-models.md`

---

## 5. 钩子疲劳

**问题**：结尾的钩子类型是不是和前几章重复了？

**检查方法**：
- 查看 `chapter_summaries.md` 最近3-5章的钩子类型列
- 同一钩子类型是否连续出现2次以上？
- 是否某一种钩子类型用得明显比其他的多？

**直觉验证**：作为读者——最近几章的结尾，你是不是已经能猜到"又是这种结尾"？

**行动建议**：
- 同类型连续 → 下一章换类型。参考 `hook-types.md` 的类型推荐配比。
- 某类型过载 → 检查是否过度依赖某一种钩子。特别是"悬念断裂"——最强但也最容易滥用的钩子。

---

## 6. 伏笔密度

**问题**：现在有多少未回收的伏笔？

**检查方法**：
- 打开 `pending_hooks.md`
- 数一数当前状态为"未回收"或"进行中"的伏笔有多少条
- 超过12条 → 是否有伏笔已经太久（>30章）没有推进？

**直觉验证**：作为作者——你还记得每一个埋下去的伏笔吗？有一条你都忘了，读者大概率也忘了。

**行动建议**：
- ≤8条 → 健康。可以考虑埋新伏笔。
- 9-12条 → 注意。优先回收或推进现有伏笔，再考虑新埋。
- >12条 → 警告。选择：哪些可以在近期（3-5章内）回收？哪些已经不重要、可以标记为"放弃"（明确写清放弃原因）？哪些可以合并回收？
- 超过30章未推进的伏笔 → 要么近期回收，要么标记放弃。不要挂着。

---

## 使用频率

- 每写完10-15章跑一次即可
- 不需要每章都跑（诊断是为了发现"慢问题"，不是"快bug"）
- 如果某一项反复出现同样的提示 → 可能不是"该章的临时问题"，而是写作习惯的系统性偏向

---

## 与QC的区别

| | QC（质检） | Novel Doctor（诊断） |
|---|---|---|
| 频率 | 每章 | 每10-15章 |
| 检测对象 | 单章 | 多章（趋势/模式） |
| 方式 | 脚本+人工 | 纯人工直觉 |
| 结果 | PASS/FAIL | 观察+建议 |
| 目的 | 防止低级错误 | 发现写作习惯的系统性偏向 |
