# 5层质检体系

> 写完整章后逐层过滤。低层级靠脚本自动化，高层级靠人工直觉。熔断机制保护写作节奏。

---

## 4a. 自动脚本检测（2秒·PASS/FAIL）

运行 `quality-check.sh`，检测7项指标（3项硬性FAIL + 4项弹性WARNING）：

| # | 检测项 | 阈值 | 类型 |
|---|--------|------|------|
| 1 | 破折号/省略号数量 | 20-35个/章（通讯章/内心独白章≤50） | ⚠️弹性警告 |
| 2 | "不是X是Y"句式计数 | ≤2次/章 | 🔴硬性FAIL |
| 3 | 抽象情绪词计数 | 0个（基线） | 🔴硬性FAIL |
| 4 | 正文字数 | 1500-5000字 | ⚠️弹性警告 |
| 5 | 一句话段落数量 | ≥5个 | ⚠️弹性警告 |
| 6 | 最长段落字数 | ≤250字 | 🔴硬性FAIL |
| 7 | 连续4逗号无句号段落数 | 0个 | ⚠️弹性警告 |

**硬性FAIL**（#2/#3/#6）不达标→修改→最多3轮。第3轮仍未通过→标记[QC-FAIL]→人工裁决，不阻塞流水线。
**弹性WARNING**（#1/#4/#5/#7）超阈值→输出⚠️警告→作者人工判断是否修改。不自动FAIL。

> 各类型阈值考量见 `references/writing-rules.md`（建议，不强制覆写）。

## 4a2. 韵律粗检（30秒·非硬性·仅供参考）

在 4a 通过后进行。两项统计输出，不做 PASS/FAIL：

### 连续短句计数
- 扫描本章，数出连续 ≤8 字的句子组（至少连续3句）
- 输出：短句组数量 + 位置（段落号）
- 最优区间：2-5组/千字
- 过多 → 碎片化节奏
- 过少 → 缺少节奏变化

### 段落长度分布
- 计算所有段落长度的标准差
- 标准差 < 40 → 标记 [节奏-平头]（段落长度过于均匀，可能缺少节奏变化）
- 标准差 > 80 → 标记 [节奏-剧烈]（段落长度极不均匀，检查是否有意为之）
- 不输出 PASS/FAIL，仅输出统计值供作者参考

`quality-check.sh` 新增这2项统计输出（仅统计，不判定通过/失败）。

## 4b. 命名一致性（30秒·人工）

快速扫本章主要人名/地名，确认无意外变体：

- "秦队"→"秦队长"（如非有意为之 → 修正）
- "祺"→"祁"（同音错字）
- 地名全称/简称混用（"长乐区" vs "长乐" — 确认一致性）
- 同一角色两个称呼在叙述中混用（"林哲" vs "小林" — 确认是否有意）

**只需30秒**。不是仔细审校——快速扫一遍发现明显不一致。

## 4c. 桥段重复检测（30秒·人工）

对照 `chapter_summaries.md` 最近20章的 [桥段类型] 标签：

- 确认本章无显性重复（如连续两章都有"丧尸群围攻超市"）
- 如果有相似桥段，确认本章有足够的差异化（不同场景/不同角色/不同结果）
- 桥段标签应从 chapter_summaries.md 的 L1 冻结摘要中提取

**只需要确认"显性重复"**——完全相同的场景类型+相同的场景结构。类似但不同的不需要标记。

## 4d. 物理/数字/元叙事自洽（30秒·人工）

扫描本章：

### 数字自洽
- 叙述中声称的数量（"四个字""三个人""五公里"）与实际是否一致
- 示例：如果写"他说了四个字"但后面只有三个字 → 标记

### 物理约束
- 车辆载客量/空间容量是否合理
- 时间距离是否符合物理现实
- 参考 `current_state.md` 中的物资/载具登记进行对照

### 元叙事污染
- 正文/对话中是否出现"第X章""前文提到""如前所述""在这一章里"等用语
- 如发现 → 删除。这些词破坏了叙事沉浸。

## 4e. 核验 + 防退步（30秒·PASS/FAIL）

> v10.3 从3项核验升级为6项，新增核型检查/感情线距离/防退步公式高亮。

**检查清单**：
- [ ] **① 核型检查：每个场景的角色内心运动是否清晰可辨（变化型/失败型/深化型）？** 遮住描写+动作，仅看行为/对话/身体反应——能看出属于哪种核型吗？不能→未找到核。
  - 🔄 变化型：从X到Y（明确的变化）
  - ❌🔄 试图变化但失败型：试图从X到Y，卡在中间
  - ⬇️ 深化型：从X到更深的X（不是变化，是加深）
- [ ] **② 感官载体独特性**。载体是否≤2个且只有该角色才会注意到？如果是"通用感官"（任何人都会看到/听到的）→未找到真正的载体。例外：两种感官对比/接力时允许2个。
- [ ] **③ 公式检测**。连续3次"他没说话/她没回答/他坐了很久/她看了很久"→至少2处重写。**解释性收尾**（"意思就是……/他知道……/她明白……"）→删除。这些句子的功能是把读者的感受替作者说出来——信任读者，不要替他们总结。
- [ ] **④ 感情线核验（如适用）**。本章有感情互动场景→结束后心理距离近了还是远了？如果没变→缺核。确认关系后的感情场景不是"安全区"——每场必须推进或后退一步，不能停在原地。
- [ ] **⑤ 如有NPC死亡**，是否满足铁律①②③④？
  - [ ] 生前回响（习惯/物品/承诺）
  - [ ] 主角特写（至少一个直接反应镜头）
  - [ ] 错动作（至少一个主角做错一个动作）
  - [ ] 禁止"一群人冲上去喊名字"+禁止模糊人称
- [ ] **⑥ 防退步：运行公式信号词高亮**。扫描全章标记所有"仿佛/似乎/在...中/感到/觉得/知道/认为/明白/他没说话/她看了很久/意思就是/他知道/她明白"。每个高亮位置确认：是有意识选择还是肌肉记忆？至少2处公式信号词必须重写。

**判定**：
- 全部通过 → PASS
- 任一项未通过 → FAIL → 退回修改（与4a同等待遇，3轮熔断）

**参考文件**：`references/nucleus-first-method.md`

## 熔断机制

```
4a FAIL → 修改 → 4a FAIL → 修改 → 4a FAIL
    ↓
第3轮仍未通过 → 标记 [QC-FAIL] + 记录未通过项 → 交作者人工裁决
    ↓
流水线不阻塞 — 继续 4a2/4b/4c/4d
```

**设计原理**：QC 是辅助工具，不是写作的障碍。如果一条规则反复导致失败——可能是规则本身在该场景下不适用。作者有最终裁决权。
