# Agent Novel Skill v9.0 — 中文详细文档

> Claude Code 万能小说创作技能。一个想法 → 4问 → 自动生成全项目 → 写到完结。78 章连载实战踩坑 → 万能化升级。

---

## 这是什么

Agent Novel Skill 是一份 Claude Code 的 skill 定义文件。放到 `~/.claude/skills/agent-novel/` 目录下，在 Claude Code 中输入 "开始新小说项目"，skill 自动引导 Bootstrap 流程——4个问题 → 自动生成全部项目文件（MASTER_SETTING + 6个追踪文件 + CLAUDE.md + .project.md）。然后说 "写第1章"，6条硬性规则 + 写作技法库 + 5层质检 + L1 冻结摘要，帮你完成从规划到质检的完整一章写作流程。

**核心定位**：不是 AI 代笔。AI 写初稿，你做决定。这个 skill 的作用是——让 AI 写的初稿不那么"AI"，让你做决定时有数据支撑。

**v9.0 万能化**：支持任意类型（奇幻/都市/古风/科幻/悬疑/末世/言情/历史/武侠/轻小说）。规则基线不变，各类型调整建议在 `references/` 中按需读取——建议，非强制，作者裁决。

## 为什么你需要这个 Skill

### 问题：AI 写小说有三大痛点

1. **AI 腔** — "他感到一阵恐惧""仿佛整个世界都在崩塌""不是在逃避，而是在面对"。读者翻三页就弃了。
2. **低级错误** — 正文出现"第X章"、"四个字"实际五个字、5座车塞8个人。作者自己很难逐条发现。
3. **长篇失控** — 写到第 80 章，忘了第 20 章埋的伏笔。读者记得，你忘了。

### 解决方案

| 痛点 | Agent Novel Skill 的应对 |
|------|--------------------------|
| AI 腔 | 6条硬性规则（抽象情绪词禁止、"不是X是Y"限2次、段落250字上限、句式不连续复用）+ **写作技法库**（7大板块：感官/对话/节奏/叙述距离/场景结构/情感传达/描写） |
| 低级错误 | 5层质检——第1层脚本自动扫描(2秒) + 韵律粗检(30秒) + 第2-4层人工目检(各30秒) + 熔断机制保护写作节奏 |
| 长篇失控 | L1冻结摘要——每章300-500字，写完即冻，永不回改 + 6个追踪文件维护所有伏笔/角色/支线状态 |
| 不知道怎么写 | **Bootstrap**——说"开始新小说"→ 4问 → 自动生成全部项目文件，立刻开工 |

```bash
git clone https://github.com/RTY798/agent-novel.git
cd agent-novel
bash install.sh
```

`install.sh` 做了三件事：
1. 把 `SKILL.md` 复制到 `~/.claude/skills/agent-novel/`
2. 把 `references/`（9个渐进披露文件）复制到 skill 目录

不需要 pip install。不需要 npm install。不需要 Docker。零运行时依赖。

## 使用

### Bootstrap：创建新项目

在 Claude Code 对话中输入：

```
开始新小说项目
```

Skill 自动引导 4 个问题：
1. 小说标题/暂定名
2. 类型/风格
3. 一句话核心前提（可填"未定"）
4. 主角名字+一句话标签

回答完毕后，自动生成完整项目文件。说 "写第1章" 即可开始。或者说 "完善设定" 深入规划世界观/配角/伏笔。

### Write：续写章节

输入 `写第79章` 或 `写下一章`。Skill 自动加载，按 6 步流水线执行：

1. 准备 → 加载 MASTER_SETTING + current_state + 前3章 + 写作技法库
2. 规划 → L0写作意图卡 + 钩子选择 + 节奏模型选择
3. 写作 → 1800-5000字
4. 质检 → 4a脚本 + 4a2韵律粗检 + 4b命名 + 4c桥段 + 4d自洽
5. 图片 → (可选)
6. 更新 → 按序写入6个追踪文件

### 诊断

输入 `诊断小说` 或 `检查健康度`。加载 6 项人工检查清单（段落呼吸/对话叙述平衡/感官切换/情绪曲线/钩子疲劳/伏笔密度）。不靠脚本造假阳性——靠作者的直觉判断。

## 6 条硬性规则

### 1. 抽象情绪词 0 容忍
感到、觉得、知道、认为、明白 — 禁止出现在叙述中。
不告诉读者"他害怕"——让他看到手抖、听到呼吸变快、感受到后背贴墙的凉意。

### 2. "不是X是Y"句式 ≤ 2 次/章
偶尔用一次有冲击力（"不是丧尸。是人。"）。用多了就是作者在说教。

### 3. 同一角色连续说话 ≤ 3 句
超过 3 句，插入动作/环境/沉默。防止对话变成"漂浮的台词"。

### 4. 段落 ≤ 250 字
250 字 ≈ 手机屏 10 行。超过这个长度，读者开始跳读。这是生理事实，不是文学判断。

### 5. 同一句式不连续复用
"他做了A。他做了B。他做了C。"——这不是风格，是模板 bug。

### 6. 不写角色解释他人心理状态
没有人能读心。角色当然也不能。让角色用观察和猜测替代"知道"。

> 各类型的调整建议见 skill 的 `references/writing-rules.md`（建议，非强制。最终裁决权在作者。）

## v9.0 新增：写作技法库

v8.0 有规则——告诉 AI "不要写什么"。v9.0 新增写作技法库——告诉 AI "应该怎么写"。

7大板块：
1. **感官描写工艺** — 五感切换频率、感官与情绪的对应、贴着皮肤写
2. **对话工艺** — 角色声音差异化（5维度）、潜台词三层检查、沉默使用
3. **节奏控制** — 句长变化、段落节奏、场景呼吸、平头避免
4. **叙述距离控制** — 特写/中景/全景、距离切换信号
5. **场景结构** — 目标→阻碍→变化、节拍分解、删减法
6. **情感传达技法** — 身体反应替代情绪词、情感升级梯度、反向技法
7. **描写工艺** — 具体>抽象、动态>静态、细节选择标准

每次写章节时自动加载。78章实战淬炼的写作知识，不是理论推导。

## 5 层质检

```
写完整章
  │
  ├→ 4a. quality-check.sh — 7+2项自动扫描，2秒
  │     破折号数量 / "不是是"句式 / 抽象情绪词 / 字数 /
  │     一句话段落数 / 最长段落 / 连续逗号无句号
  │     + 段落标准差 / 连续短句统计（仅统计，不判定）
  │     FAIL → 修改 → 重跑（最多3轮）
  │
  ├→ 4a2. 韵律粗检 (30秒) — 段落分布+短句节奏（非硬性·仅供参考）
  ├→ 4b. 命名一致性 (30秒) — 秦队→秦队长？祺→祁？
  ├→ 4c. 桥段重复 (30秒) — 最近20章同类桥段去重
  └→ 4d. 物理/数字/元叙事 (30秒) — 5座车≠8人
```

## L1 冻结摘要

长篇连载最致命的问题：写到后面忘了前面。

**L1 冻结摘要**：每章写完后，产出 300-500 字的摘要。写定后**永不修改**。如果后续章节发现摘要中的事实错误——不在摘要里改，在 `continuity-issues.md` 里追加修正记录。

这保证了：
- 任何时候回头看摘要，看到的是"当时的理解"
- 不会因为后来的剧情调整而"改写历史"
- 修正记录本身就是连载的成长档案

## 2个通用节奏模型

替代了 v8.0 方案中的 6 个具体场景模板。覆盖 80% 场景。

- **上升型（冲突推进）**：5-6节拍，情绪持续上升。适用于战斗/对抗/揭露真相/追逐/紧张对话。叙述距离：全景→中景→特写（逐渐逼近）。
- **下降型（情感/反思）**：4-5节拍，情绪下降或U型。适用于情感爆发后的余波/角色独处/旅途过渡/安静时刻。叙述距离：特写→中景→全景（逐渐拉远）。

> 详细节拍分解见 `references/rhythm-models.md`。6个具体场景示例见 `docs/scene-examples/`（可选参考）。

## 设计原则

1. **知识库 + 建议，而非自动调节的规则引擎**。最终裁决权留给作者。
2. **最小改动拿最大收益**。不建维护地狱。
3. **每一处自动化都有明确的人和工具分工**。AI 负责写，QC 脚本负责检查，你负责裁决。
4. **规则服务于文字，不是反过来**。如果一条规则让文字更生硬——无视它。
