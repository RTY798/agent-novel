# Agent Novel Skill — 小说创作核心规则

## 6条硬性规则
1. 抽象情绪词0容忍：感到/觉得/知道/认为/明白 — 禁止出现在叙述中
2. "不是X是Y"句式 ≤ 2次/章
3. 同一角色连续说话 ≤ 3句（中间必须插入动作/环境/沉默）
4. 段落 ≤ 250字
5. 同一句式不连续复用（开头/结尾/段落结构）
6. 不写角色解释他人心理状态

## 禁用词表
- 抽象情绪：感到、觉得、知道、认为、明白
- AI腔：仿佛、似乎、在...中、不是...而是...
- 后宫/城主夫人/宠幸等依附者线禁用标签

## 叙述者模式
- 贴着皮肤写（特写）= 战斗/关键决定/身体伤害
- 退后一步写（全景）= 过渡/休整/消化期
- 记忆介入：感官描述时自然引入回忆（"这场雨让他想起…"）

## 6型章末钩子
信息炸弹 / 悬念断裂 / 情绪共振 / 反差沉默 / 动作戛然 / 命运转折
原则：避开前3章已用类型

## 爽点梯级（写爽点场景时对照）
战斗：
L1被碾压 — 压倒性劣势，主角用计谋或代价勉强脱离
L2险胜 — 实力接近，靠某个细节翻盘，主角付出可见代价
L3智取 — 以弱胜强但合情合理，读者能复盘每一步
L4碾压 — 主角优势明显，但必须有铺垫（非凭空变强）
L5不战而胜 — 通过之前的布局让他人代劳，或对方主动退让

情感：
L1被理解 — 有人说出主角自己说不出口的话
L2和解 — 破裂关系的第一次重建信号
L3回应 — 长线情感的明确正面反馈
L4清算 — 压抑关系的终极爆发或终结

## 6步流水线
1. 准备：
   - 必须加载：MASTER_SETTING + current_state + 前3章节选
   - 按需加载：其他追踪文件（清单见当前小说项目的 CLAUDE.md）
   - 裁剪规则：当上下文接近15k token时，按序裁剪——①写作指南 ②非出场角色弧光 ③关键词检索结果。绝不裁剪 MASTER_SETTING / current_state / 活跃伏笔
2. 规划：单章3问（本章核心情绪？情感锚点角色？哪些可以不写？）+ 选钩子类型 + 确认本章伏笔操作（≥1条）
3. 写作：按大纲写1800-5000字，严格遵守6条硬性规则
4. 质检：
   a. 运行 quality-check.sh，不达标则修改，最多3轮。第3轮仍未通过则标记[QC-FAIL]并记录未通过项，交作者人工裁决。不阻塞流水线。
   b. 命名一致性：快速扫本章主要人名/地名，确认无意外变体（如"秦队"写成"秦队长"且非有意为之）。
   c. 桥段检查：对照 chapter_summaries.md 最近20章的[桥段类型]标签，确认本章无显性重复。
   d. 物理/数字/元叙事自洽：扫一遍本章——
      - 数字自洽：叙述中声称的数量（"四个字""三个人""五公里"）与实际是否一致
      - 物理约束：车辆载客量/空间容量/时间距离是否符合物理现实（参考 current_state.md 物资/载具登记）
      - 元叙事污染：正文/对话中是否出现了"第X章""前文提到""如前所述"等元叙事用语
5. 图片：node scripts/convert-to-images.js（可选）
6. 更新：按序写入tracking/下追踪文件——
   ① chapter_summaries.md — 追加 [L1冻结摘要]（300-500字，写定后永不回改）+ 钩子类型 + [桥段类型]标签
   ② current_state.md — 更新角色位置/团队状态/物资/威胁
   ③ pending_hooks.md — 伏笔推进/回收/新埋
   ④ subplot_board.md — Strand Weave日志+断档计数
   ⑤ character_arcs.md — 仅当弧光状态变化时更新
   ⑥ emotional_arcs.md — 仅当感情线/爽点状态变化时更新
   如后续发现L1冻结摘要中的事实错误，在 logs/continuity-issues.md 登记，不覆盖已冻结的L1。
