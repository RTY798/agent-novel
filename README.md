```
一句话想法 → 4问 → 自动生成全项目 → 写第1章
    │
6条硬性规则 → 消灭AI腔（不靠审美·靠规则执行）
    │         + 写作技法库（感官/对话/节奏/叙述距离）
    │
5层质检 ────→ ① quality-check.sh 自动扫描(7+2项/2秒)
    │        ② 韵律粗检(30s) ③ 命名一致性(30s)
    │        ④ 桥段重复去重(30s) ⑤ 物理/数字/元叙事自洽(30s)
    │        ⑥ 熔断: 3轮未通过→标记[QC-FAIL]→人工裁决
    │
L1冻结摘要 → 每章300-500字·写定冻存·永不回改
    │         (写到200章也不会忘记第20章的伏笔)
    │
爽点梯级 → 战斗L1-L5(被碾压→险胜→智取→碾压→不战而胜)
    │     情感L1-L4(被理解→和解→回应→清算)
    │
2个通用节奏模型 → 上升型(冲突推进) / 下降型(情感反思)
    │
6型钩子  → 信息炸弹/悬念断裂/情绪共振/反差沉默/动作戛然/命运转折
    │     避开前3章已用类型·防止钩子疲劳
    │
上下文裁剪 → 15k token阈值·按优先级丢弃·绝不丢关键状态
```

**78 章连载实战踩坑提炼 → 万能化升级。** 支持任意类型：奇幻/都市/古风/科幻/悬疑/末世/言情/历史/武侠/轻小说。

```
开始新小说项目

```

然后说：

```
写第1章
```

> 想先深入规划？说 **"完善设定"** 进入第2轮——世界观/配角/伏笔/叙事视角，按需展开。

## Before / After

左边是未经 skill 处理的 AI 腔文本。右边是同一场景，经过 6 条硬性规则 + QC 过滤后的样子。

| Before | After |
|-----------|----------|
| 他感到一阵深深的恐惧从心底升起，他觉得自己可能永远也走不出这座死城了。仿佛整个世界都在崩塌。 | 林哲的后背贴着墙。水泥是凉的。他从墙角探出半个头——暗处有东西在动。 |
| 她知道他在害怕，但她同时也知道他是对的。这不是软弱的表现，而是一个人在面对巨大危险时最真实的反应。 | 苏瑶的手按在腰间的撬棍上。金属贴着她掌心的茧——硬的——锈的触感。她的呼吸从鼻子里出来——两次——然后停了。 |

> 完整 Before/After 对比见 [`demo/`](demo/)

**区别在哪？** Before 是叙述者在替角色做总结。After 是读者站在角色的肩膀上看世界。这就是 6 条硬性规则的作用——不靠审美，靠规则。

## v9.0 新增：写作技法库

v8.0 有规则——告诉 AI "不要写什么"。v9.0 新增写作技法库——告诉 AI "应该怎么写"。

| 技法板块 | 内容 |
|---------|------|
| **感官描写工艺** | 五感+内脏觉切换频率、感官与情绪的对应、贴着皮肤写 |
| **对话工艺** | 角色声音差异化（5维度）、潜台词三层检查、对话锚定、沉默使用 |
| **节奏控制** | 句长变化规律、段落节奏分配、场景呼吸、"平头"避免 |
| **叙述距离控制** | 特写/中景/全景的选择逻辑、距离切换信号、常见错误 |
| **场景结构** | 场景三要素（目标→阻碍→变化）、节拍分解、入场出场、删减法 |
| **情感传达技法** | 身体反应替代情绪词、情感升级梯度（4级）、反向技法 |
| **描写工艺** | 具体>抽象、动态>静态、细节选择标准、描写的节奏功能 |

> 每次写章节时自动加载。规则 + 技法 = 完整的写作质量体系。

## 这 78 章中我们踩过的 5 个经典坑

| 坑 | 表现 | Skill 如何防 |
|----|------|-------------|
| **元叙事污染** | 角色说"这是第12章给你的" | QC 4d 扫描"第X章""前文提到"等模式 |
| **数字不自洽** | "四个字"实际是五个字 | QC 4d 数字声称 vs 实际逐条比对 |
| **物理穿帮** | 5座比亚迪塞了8个人 | current_state.md 维护载具容量，QC 4d 对照检查 |
| **伏笔蒸发** | 40章后忘了20章的约定 | L1 冻结摘要——每章写定冻存，永不丢失 |
| **钩子疲劳** | 连用三次同类型章末钩子 | 6型钩子分类 + 避开前3章已用类型 |

> 详细踩坑集：[`docs/real-pitfalls.md`](docs/real-pitfalls.md)

## What It Does

```
写第X章
  │
  ├→ 1. 准备  加载 MASTER_SETTING + 世界快照 + 前3章语感 + 写作技法库
  │           上下文接近15k token时按优先级裁剪
  │
  ├→ 2. 规划  L0写作意图卡（核心情绪+情感锚点+节拍预排4-6+删减清单）
  │           选钩子类型（避开前3章已用）+ 节奏模型（上升型/下降型）
  │
  ├→ 3. 写作  1800-5000字 · 6条硬性规则 + 写作技法库
  │
  ├→ 4. 质检  4a: quality-check.sh 自动扫描 (7+2项, 2秒)
  │           4a2: 韵律粗检 (段落分布+短句统计, 30秒)
  │           4b: 命名一致性目检 (30秒)
  │           4c: 桥段重复目检 (30秒)
  │           4d: 物理/数字/元叙事自洽 (30秒)
  │           不达标→修改→最多3轮→标记[QC-FAIL]→人工裁决
  │
  ├→ 5. 图片  node scripts/convert-to-images.js (可选)
  │
  └→ 6. 更新  按序写入6个追踪文件
              ① chapter_summaries (L1冻结摘要+钩子+桥段标签)
              ② current_state (角色位置/物资/威胁)
              ③ pending_hooks (伏笔推进/回收/新埋)
              ④ subplot_board (支线断档计数)
              ⑤ character_arcs (弧光变化时)
              ⑥ emotional_arcs (感情线变化时)
```

## 6 大卖点

### 1. 极简 Bootstrap —— 一句话想法到完整项目

说"开始新小说" → 回答4个问题（标题/类型/前提/主角）→ **自动生成全部项目文件**。想要更细？说"完善设定"深入规划。两种路径，互不阻塞。

### 2. 写作技法库 —— 不只是规则

7大技法板块（感官/对话/节奏/叙述距离/场景结构/情感传达/描写）。规则告诉 AI "不要写什么"，技法告诉 AI "应该怎么写"。78章实战淬炼的写作知识，不是理论推导。

### 3. 6 条硬性规则 —— 消灭 AI 腔

不靠审美判断，靠规则执行。每条规则都有对应的检测方式。详见 [`docs/6-rules.md`](docs/6-rules.md)

### 4. 5 层质检 —— 低级错误归零

- **4a** (自动): quality-check.sh — 7+2项指标，2秒跑完
- **4a2** (30s): 韵律粗检 — 段落分布+短句统计（非硬性，仅供参考）
- **4b** (30s): 命名一致性 — 秦队→秦队长？祺→祁？
- **4c** (30s): 桥段重复 — 最近20章同类桥段去重
- **4d** (30s): 物理/数字/元叙事自洽 — 5座车坐不了8个人
- **熔断**: 3轮未通过 → 标记 [QC-FAIL] → 人工裁决

详见 [`docs/qc-system.md`](docs/qc-system.md)

### 5. L1 冻结摘要 + 6 文件追踪 —— 长篇不失控

每章300-500字摘要，写定冻存，永不回改。6个追踪文件维护所有伏笔/角色/支线状态。写到200章也不会忘记第20章的约定。

### 6. 类型全覆盖 —— 建议而非规则引擎

支持10种类型。每种类型的调整建议（规则参数/禁用词/写作要点）在 `references/` 中按需读取。**不自动覆写参数——最终裁决权在作者。**

## 这个 Skill 是什么 / 不是什么

**是**：
- 写作质量控制系统（规则过滤 + QC 检测 + 技法支撑）
- 长篇连载项目管理工具（Bootstrap 生成 + 6 文件追踪 + L1 冻结摘要）
- 写作技法知识库（7大板块，78章实战淬炼）

**不是**：
- 不是一键生成整本小说的"AI 代笔"——AI 写初稿，你做决定
- 不是"只有末世公路才能用"——v9.0 支持任意类型，靠建议不靠参数覆写

## 适用场景

- 长篇连载（≥10章），需要维护伏笔/支线/角色弧光的复杂叙事
- 任意类型：奇幻/都市/古风/科幻/悬疑/末世/言情/历史/武侠/轻小说
- 8人以下主角团，章节长度 1800-5000 字
